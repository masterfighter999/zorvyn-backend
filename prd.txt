Finance Data Processing & Access-Control Backend (FastAPI Architecture)
1. Product Vision

Build a backend platform for a finance dashboard that:

manages financial transactions
enforces role-based access control
exposes summary analytics
guarantees reliable write processing
prevents duplicate writes
supports asynchronous event pipelines
remains resilient under dependency failures
tracks user activity and system access

The backend should demonstrate production-grade backend engineering patterns while remaining clean and maintainable.

6. Redis Design
6.1 Idempotency
Header Required
Idempotency-Key
Redis Key
idempotency:{key}
Flow
Request arrives
↓
Check Redis
↓
Exists → return stored response
↓
Not exists → process request
↓
Store result
6.2 Cache Aside Pattern
Cached APIs
dashboard summary
trends
categories
Redis Keys
dashboard:summary:{user_id}
dashboard:trends:{month}
Flow
Check cache
↓
Miss → DB query
↓
Store cache
↓
Return response
Invalidation

On record mutation:

delete dashboard:summary:{user_id}
6.3 Distributed Locks / Mutex

Used for concurrent financial updates.

Redis Lock Key
lock:record:{id}
Flow
Acquire lock
Process transaction
Release lock

This prevents race conditions.

10. Circuit Breakers
Redis Circuit Breaker

If Redis unavailable:

fallback to DB.

Behavior
cache fail → DB query
Kafka Circuit Breaker

If Kafka unavailable:

outbox retains event.

Behavior
publish fail → retry later
